SimComm: SimComm, SimComm-package

SimComm

Community simulator.

Details

This package alows simulating the evolution along time of communities of agents, whose location is decribed by a pattern (a matrix or a point pattern). The communities are represented by R6 classes: community_model. A model contains a pattern, a timeline and an evolution method that is run along the timeline. This is a classical approach in community ecoology modeling or agent based models.

Abstract classes

community_model

Community Model Class

The model represents a community, i.e. a set of interacting objects called agents. Their location is described by a pattern that evolves along a timeline.

Usage

Format

R6Class object.

Value

An R6Class.

Fields

The pattern which describes the location of agents.

timeline
A numeric vector.

last_time
The last time (in the time line) the model has been run.

The past patterns of the model, obtained by run and saved.

cm_Conway

Conway’s game of life

A community_matrixmodel where each cell contains or not an individual. At each generation, an individual may survive or not and empty cells be filled by a new individuals.

Usage 

cm_Conway

Format

An object of class R6ClassGenerator of length 24.

Value

An R6Class.

Details

The survival and generation rules are fixed by the number of neighbors of each cell. Edge effects are eliminated by a toroidal correction.

The number of neighbors necessary for an individual to survive.

to_generate
The number of neighbors necessary for an empty cell to generate an individual.

patterns

Patterns

The location of the agents of a model is described by a pattern. Several patterns are available by default. They may be point patterns (of class ppp). Regular point patterns are called grids. They may also be matrices with no explicit coordinates by only notions of neighborhood. Matrix models are by far faster to run. Patterns are to be used in the pattern field of a community_model.

Usage 

pattern_grid(nx = 8, ny = nx, S = 300, Distribution = "lnorm", sd = 1, prob = 0.1, alpha = 40)


pattern_matrix_individuals(nx = 8, ny = nx, S = 300, Distribution = "lnorm", sd = 1, prob = 0.1, alpha = 40)


pattern_matrix_logical(nx = 8, ny = nx, prob = 0.5)


pm_Conway_oscillator()


pm_Conway_glider(nx = 50, ny = nx)

Arguments

nx
The number of X values (columns) of the grid or the matrix.
ny
The number of y values (lines) of the grid or the matrix.
S
The number of species.
Distribution
The distribution of species frequencies. May be “lnorm” (log-normal), “lseries” (log-series), “geom” (geometric) or “bstick” (broken stick).
sd
The simulated distribution standard deviation. For the log-normal distribution, this is the standard deviation on the log scale.
prob
The proportion of ressources taken by successive species in the geometric model.
alpha
Fishers alpha in the log-series model.

Value

evolve methods should modifiy the models pattern directly and return NULL.

Details

Available patterns:

pattern_grid
A rectangular, regular grid of points. Each point has marks. Point are on a regular grid of 1x1 units of distance.

pattern_matrix_individuals
A matrix. Each cell of the matrix is an agent and has a numeric value equal to its type.

pattern_matrix_logical
A matrix. Each cell of the matrix is FALSE or TRUE (dead or alive, present or absent…).

Diversity.pattern_matrix_individuals

HCDT diversity of a matrix of individuals

Diversity calls Tsallis to calculate entropy and transforms it into diversity by calculating its deformed exponential.

Usage 

"Diversity"(NorP, q = 1, Correction = "Best", ..., CheckArguments = TRUE)

Arguments

NorP
An object of class “wmppp” (pattern_matrix_individuals).
q
A number: the order of entropy. Some corrections allow only a positive number. Default is 1 for Shannon entropy.
Correction
A string containing one of the possible corrections: “None” (no correction), “ChaoShen”, “GenCov”, “Grassberger”, “Holste”, “Bonachela”, “ZhangGrabchak”, or “ChaoWangJost”, “Marcon”, “UnveilC”, “UnveiliC”, “UnveilJ” or “Best”, the default value. Currently, “Best” is “ChaoWangJost”.
…
Further arguments. Unsused.
CheckArguments
If TRUE (default), the function arguments are verified. Should be set to FALSE to save time in simulations for example, when the arguments have been checked elsewhere.

Value

A named number equal to the calculated diversity. The name is that of the bias correction used.
